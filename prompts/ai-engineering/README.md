## About This Category

This category focuses on real-world problems related to AI systems, engineering decisions, and practical constraints.

These problems test:
- System thinking
- Trade-offs
- Collaboration
- Decision-making under uncertainty

---

## What Teams Are Expected To Do

Teams should:
- Pick ONE problem prompt
- Understand the real-world context
- Discuss possible approaches
- Make justified decisions
- Document their reasoning clearly

You are **not expected** to build a perfect or complete system.

---

## What Is Being Evaluated

Evaluation is **not about code quantity**.

It focuses on:
- How well the problem is understood
- How decisions are explained
- How the team collaborates
- How trade-offs and risks are handled

---

## Team Requirement

❗ Minimum team size: **5 members**  
❗ Solo work is not reviewed

---

## How To Proceed

1. Read the prompts listed below
2. Form your team
3. Create a separate public repository
4. Follow the standard README format
5. Begin collaborative work
